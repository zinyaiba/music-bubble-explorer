# Firebase Analytics クイックスタート

## 📊 使用状況を確認する3つのステップ

### 1. Firebase Consoleにアクセス

1. [Firebase Console](https://console.firebase.google.com/)を開く
2. プロジェクト「music-bubble-explorer」を選択
3. 左メニューから「Analytics」をクリック

### 2. リアルタイムユーザーを確認

**ダッシュボード画面で:**
- 現在のアクティブユーザー数
- 過去30分のユーザー数推移
- 使用されているデバイス（モバイル/デスクトップ）

### 3. 詳細な分析を見る

**イベント画面で:**
- どの機能が最も使われているか
- シャボン玉のクリック数
- 楽曲・タグの登録数
- 詳細表示の閲覧数

## 📈 よく見る指標

### アクティブユーザー数
- **今日**: 本日のユニークユーザー数
- **7日間**: 過去7日間のアクティブユーザー
- **30日間**: 過去30日間のアクティブユーザー

### 人気機能ランキング
1. **bubble_click**: シャボン玉のクリック数
2. **song_detail_view**: 楽曲詳細の閲覧数
3. **tag_detail_view**: タグ詳細の閲覧数
4. **song_registration**: 楽曲登録数
5. **tag_registration**: タグ登録数

### デバイス別利用状況
- モバイル vs デスクトップの比率
- 最も使われているブラウザ
- OSの分布

## 🎯 分析のヒント

### ユーザーエンゲージメントを測る
- **平均セッション時間**: ユーザーがどれくらい滞在しているか
- **セッションあたりのイベント数**: どれくらいアクティブに使っているか
- **リテンション率**: 何人が再訪問しているか

### 機能の人気度を比較
- イベント画面で各機能の使用回数を比較
- どの機能が人気か、どの機能が使われていないかを把握

### 問題の早期発見
- エラーイベントの発生頻度をチェック
- 特定の機能でエラーが多発していないか確認

## 📱 モバイルアプリでも確認可能

Firebase Analyticsのモバイルアプリをインストールすると、スマホからでも確認できます:
- [iOS版](https://apps.apple.com/app/firebase-console/id1230810726)
- [Android版](https://play.google.com/store/apps/details?id=com.google.firebase.console)

## 💡 Tips

### データの更新頻度
- **リアルタイムデータ**: 数秒で反映
- **イベント集計**: 数時間で反映
- **詳細レポート**: 最大24時間

### データの保持期間
- Firebase Analyticsは自動的に14ヶ月間データを保持
- BigQueryにエクスポートすれば無期限保存可能

### プライバシー
- 個人を特定できる情報は収集していません
- IPアドレスは自動的に匿名化されます
- ユーザーは追跡を拒否できます（ブラウザ設定）
